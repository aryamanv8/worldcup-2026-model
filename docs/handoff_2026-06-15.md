## 1. What this project is (30-second version)

Three-stage project by Aryaman Verma (CMU, applied math + computational finance):

- **Stage 1** — calibrated Poisson + Dixon-Coles match model for WC 2026, trained on ~25k international matches (2000–2026), Elo + squad value + form features, single temperature parameter T=0.77. Model is **frozen** for the tournament duration.
- **Stage 2** — used the model as a fair-value reference vs Kalshi prediction markets. Finding: no tradeable edge on liquid outrights. Market favorite-longshot bias is real but not bankable on liquid contracts.
- **Stage 3** — live $500 paper-trading experiment on per-match Kalshi markets. Quarter-Kelly sizing, 10% per-position cap, 20% max deployment, entry requires reliable zone (Elo gap −50..+150) AND ≥3¢ net edge after fees.

Full methodology in `docs/technical_record.md`. Repo: `github.com/aryamanv8/worldcup-2026-model`.

## 5. Strategic decisions made this session — do not re-litigate

These were discussed at length and decided. The next session should treat them as settled:

**Don't touch the open positions.** No hedging, no early exit. The experiment is hold-to-settlement by design. Attribution would be meaningless otherwise.

**The trading strategy has not been backtested as a strategy.** The model was backtested as a forecasting model (OOS log loss, tournament-level calibration). But the trading strategy — "bet when model FV > Kalshi ask by ≥3¢ in the reliable zone" — has never been tested against historical market prices. We validated the model. We didn't validate whether that model generates profitable trades.

**The model improvement that actually helps trading is not more calibration work.** The model is as good as it can get from historical data alone. The one thing that would genuinely help is a real-time lineup check — verify the expected XI is playing before entering a position. The model has no injury/suspension/rotation awareness. This is not a model rebuild; it's a pre-trade filter.

**New positions going forward should not all be favorite-fade.** The three open positions are already fully correlated. If the backtest is ambiguous and the open trades suggest mixed signals, the next trades should include some "favorite-boost" cases (where model agrees with the market favorite but thinks the market is still underpricing them). These are a genuinely different signal.

## 6. The priority task for the next session: historical strategy backtest

This is the most important missing piece. Before adding more paper trades, validate that the strategy — not the model, the *strategy* — would have been profitable at prior World Cups.

### What to build

A script (suggest `scripts/29_backtest_trading_strategy.py`) that:
1. Loads `data/processed/backtest_predictions_recalibrated.parquet` — this already contains the frozen model's W/D/L probabilities for 2010, 2014, 2018, 2022 WC group stage matches (recalibrated at T=0.77)
2. Fetches historical closing odds for those matches from football-data.co.uk
3. For each match: applies the same entry filters (reliable zone, ≥3¢ net edge after fees)
4. Simulates P&L under quarter-Kelly sizing
5. Reports: total trades, win rate, total P&L, P&L by tag (favorite-fade vs favorite-boost vs neutral), Sharpe-equivalent

### Where to get historical odds

`football-data.co.uk` has free CSVs with historical match odds. For World Cups:
- 2018: `https://www.football-data.co.uk/new/WC2018.csv` (or similar path — check their site)
- 2022: `https://www.football-data.co.uk/new/WC2022.csv`

The CSVs typically include columns like `B365H`, `B365D`, `B365A` (Bet365 home/draw/away odds) and `MaxH`, `MaxD`, `MaxA` (best available odds). Use closing odds (not opening) if available — these are the sharpest prices.

Convert decimal odds to implied probability: `p = 1 / decimal_odds`. These will be slightly over 100% due to bookmaker margin — de-vig the same way as in script 23.

### What's already available

`data/processed/backtest_predictions_recalibrated.parquet` has columns:
- `date`, `home_team`, `away_team`, `tournament`, `wc_year`
- `p_home_recal`, `p_draw_recal`, `p_away_recal` (the recalibrated probabilities)

The Elo history is in `data/processed/elo_history.parquet` — needed to compute Elo gaps at match time (not current Elo, which is what `team_features.parquet` has).

### Naming caveat (important)

The backtest predictions use the `martj42/international_results` naming: "Turkey" not "Turkiye", "Iran" not "IR Iran", "Czech Republic" not "Czechia", "DR Congo" not "Congo DR". Football-data.co.uk will likely use different names again. Build a crosswalk.

### What a positive result looks like

If the strategy applied to 2018 + 2022 group stages (roughly 64 group matches × 2 = ~128 matches, maybe 30-50 qualifying trades after filters) shows positive expected value after fees, that's a meaningful foundation. If it's flat or negative, the live experiment is testing noise and position sizes should stay minimal for the rest of the tournament.
